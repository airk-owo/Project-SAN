# Fix: mass-trick demo responses use the correct card type

อ่านอันนี้ก่อนนะ 💛 (read this first) — สรุปสั้น ๆ ว่าแก้อะไร ทำไม และทดสอบยังไง.

## ปัญหา (the bug)

ในหน้า **local test** (`/game/local`) ตอนเล่นการ์ดอุบายแบบ "ใส่ทุกคน" (mass trick)
ปุ่มตอบสนองของฝ่ายตรงข้าม (Demo response) ถูก **ฮาร์ดโค้ดเป็น "หลบ" เสมอ**:

- มันเรียก `playDodge` / `respondToAttack` และหาเฉพาะการ์ด `dodge`
- มันเขียนว่า **"Demo: ใช้ หลบ"** ตลอด

แต่การ์ดแต่ละใบต้องการการตอบสนองคนละแบบ:

| การ์ด | ชนิด window | ต้องตอบด้วย |
|---|---|---|
| **คนเถื่อนบุกรุก** | `mass_attack` | **โจมตี** (Attack) |
| **เกาทัณฑ์พันดอก** | `mass_dodge` | **หลบ** (Dodge) |

เพราะงั้นตอนเล่น **คนเถื่อนบุกรุก** ปุ่มเลยทั้ง **กดไม่ได้** (เพราะมันหาไพ่ `dodge`
แต่จริง ๆ ต้องใช้ `attack`) **และป้ายชื่อผิด** → mass-attack ไขไม่ออกในเดโม.

> เครดิตคนเจอบั๊ก: เล่นเทสต์แล้วสังเกตเองว่า "การ์ดนี้น่าจะต้องใช้ไพ่โจมตี ไม่ใช่ไพ่ป้องกัน" — ถูกเป๊ะ ✅

## สิ่งที่แก้ (the fix)

- เพิ่ม `resolveMassDemo()` ที่ **เลือกการ์ดที่ถูกต้องตามชนิดของ window**
  (`mass_attack` → `attack`, `mass_dodge` → `dodge`)
  แล้วเรียกฟังก์ชันของ engine ที่ถูกต้อง: `playMassResponseCard` / `declineMassResponse`.
- ปุ่มตอบสนองฝั่งคู่ต่อสู้ตอนนี้ **แตกตามชนิด window** และโชว์ป้ายให้ถูก:
  - `mass_attack` → **"Demo: ใช้ โจมตี"**
  - `mass_dodge` → **"Demo: ใช้ หลบ"**
  - กรณีปกติ (single-target attack) → **"Demo: ใช้ หลบ"** เหมือนเดิม
- ปุ่ม "ไม่ทำอะไร" ของ mass trick ตอนนี้บอกชัดว่า **เสีย 1 HP**.

**ไฟล์ที่แก้: แค่ `apps/web/app/game/local/page.tsx` เท่านั้น.**
**ไม่ได้แตะ engine (`packages/game`) เลย** — เป็นช่องว่างของหน้าเทสต์อย่างเดียว
(engine มี `playMassResponseCard` + `declineMassResponse` พร้อมอยู่แล้ว แค่หน้า demo ยังไม่ได้ต่อสาย).

## วิธีทดสอบ (how to verify)

```
npm run check -w @wtk/web    # tsc ผ่าน ไม่มี error  ✅
npm run dev                  # เปิด http://localhost:3000/game/local
```

1. กด **เริ่มใหม่** → กด **จั่ว 2 ใบ**
2. เล่น **คนเถื่อนบุกรุก** → ปุ่มต้องขึ้น **"Demo: ใช้ โจมตี"** และ **กดได้**
3. เล่น **เกาทัณฑ์พันดอก** → ปุ่มต้องขึ้น **"Demo: ใช้ หลบ"**
4. กด "ไม่ทำอะไร" บน mass trick → ผู้ตอบเสีย **1 HP**
5. โจมตีเดี่ยวปกติ → ยังขึ้นปุ่ม **หลบ** เหมือนเดิม (ไม่กระทบของเก่า)
